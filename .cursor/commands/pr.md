Prepare a pull request for the current changes.

Steps:

1. Inspect `git status`.
2. Inspect `git diff`.
3. Summarize the changed files.
4. Identify the purpose of the change.
5. Suggest a concise commit message.
6. Draft a pull request title and description.
7. Include:
   - Summary
   - Motivation
   - Changes made
   - Validation performed
   - Risks / follow-up work
8. Do not commit, push, or create the PR unless explicitly instructed.

Output format:

- Changed files
- Commit message
- PR title
- PR description
- Validation checklist
- Risks
