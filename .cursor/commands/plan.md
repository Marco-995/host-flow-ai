Create an implementation plan before making changes.

Steps:

1. Understand the requested task.
2. Inspect the relevant files and call sites.
3. Summarize the current behavior.
4. Identify affected files.
5. Propose a focused implementation approach.
6. List architectural implications.
7. List risks and edge cases.
8. List validation steps.
9. Do not edit files yet.
10. Wait for approval before implementation.

Output format:

- Task summary
- Files inspected
- Current behavior
- Proposed implementation
- Risks / edge cases
- Validation plan
- Open questions
